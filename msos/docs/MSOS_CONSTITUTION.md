# MSOS Constitution

## Version

`0.1.0`

## Purpose

The MSOS Constitution defines the engineering and operating principles for Mr Sotong Operating System.

## Principles

1. Production must never be modified directly.
2. Design must happen before implementation.
3. Testing must happen before deployment.
4. Every change must include a version, changelog, and rollback plan.
5. Notion is the single source of truth.
6. Existing components must be reused whenever practical.
7. Simplicity is preferred over unnecessary complexity.
8. Important decisions must be documented.
9. Unclear requirements must be clarified before implementation.
10. Stability, scalability, and maintainability are the default engineering standards.

## Change Control

Any change to this constitution requires:

- A version update.
- A changelog entry.
- A rollback plan.
- A linked Notion decision record.
